Overview

基于 llama.cpp 的 ComfyUI 本地 LLM / VLM 推理插件

通过自编译的 Vulkan 版 llama-cpp-python 在 ComfyUI 中运行 GGUF 大模型, 提供文本, 图片, 视频帧, 音频四种模态的推理节点, 以及围绕 LLM 结构化输出的 JSON / BBox 后处理工具链. GPU 加速走 Vulkan 路径, 与 PyTorch 的 CUDA / ROCm 完全独立. 全部用户可见文案与控制台日志经 i18n 文案层提供 zh-CN / en-US 双语.

类型隔离设计

LLM Model Loader 输出 LLAMACPPLLM, 只能连 Text Instruct; VLM Model Loader 输出 LLAMACPPVLM, 只能连 Image / Video / Audio Instruct. 连错在连线阶段即被 ComfyUI 类型系统拦截, 两种配置底层共用同一条加载路径.

采样参数节点输出 LLAMACPPARAMS, 是全部 Instruct 节点的可选输入.

已知约束

  • 单模型实例: 全局单例存储, 不支持同时加载多个模型(多组 loader+instruct 会按需切换重载).
  • 有独显时核显不可选: 与 llama.cpp 设备收集规则对齐; 强制核显需在进程启动前设 GGML_VK_VISIBLE_DEVICES.
  • import 时初始化 Vulkan: 设备枚举在插件加载期同步执行(约几百 ms), 为 UI 下拉框提供设备列表, 属有意设计.
  • mmproj 不跟随显式选卡: mtmd 只有 use_gpu 布尔开关, 无设备索引字段(上游限制); 纯 CPU 或预算装不下 mmproj 时传 False 让其留在 CPU.
  • 多分片 GGUF 不支持: 显存折算只计所选文件体积, 层数却是全模型层数, 会高估 n_gpu_layers; 请先用 llama-gguf-split --merge 合并为单文件.
