00数据流总览

实线框 = 本插件节点；虚线框 = ComfyUI 原生或第三方节点；高亮框 = 推理节点

类型隔离：llm Model Loader 输出 LLAMACPPLLM 只能连 text Instruct；vlm Model Loader 输出 LLAMACPPVLM 只能连 image / video / audio Instruct。Parameters 可接全部 4 个推理节点（图中只画了代表性连线）； 4 个推理节点的输出均为 STRING，都可接任意文本工具节点。Unload Model 为万能透传节点（any 进 any 出），可串接任意连线，未在图中展示。

01用户可见的功能

用户在 ComfyUI 前端可以直接感知、操作的能力

按模态拆分的节点体系 (v2.0)

  • 双加载器：llm Model Loader 只有 4 个字段（纯文本模型）；vlm Model Loader 8 个字段且 mmproj / chat_handler 必选，配对错误在校验期拦截
  • 类型隔离：llm 配置（LLAMACPPLLM）只能连 text Instruct，vlm 配置（LLAMACPPVLM）只能连 image/video/audio Instruct，连错线在前端直接接不上
  • 四个推理节点：text（prompt 改写等）、image（逐张 / batch 合并）、video（帧批次均匀抽帧）、audio（ASR / omni）
  • 预设按模态过滤：每个推理节点的 preset_prompt 下拉框只展示相关任务模板（如 Summarize Video 只在 video 节点出现）
  • 任务预设全中文：预设名与模板均为中文，@ 占位符按模态替换为 图像/视频/音频，audio 默认预设为"常规 - 转写"

模型推理（核心能力）

  • 在 ComfyUI 内原生运行 GGUF 格式 LLM/VLM，Vulkan 后端 GPU 加速，AMD / NVIDIA / Intel 全支持，无需 CUDA/ROCm 工具链
  • 30 种 VLM chat handler：Qwen2.5/3/3.5/3.6-VL、Gemma3/4、GLM-4.6V/4.1V、MiniCPM v2.6/4.5/4.6、LLaVA 系、LFM2/2.5-VL、Step3-VL，以及 OCR/文档专用（DeepSeek-OCR、PaddleOCR-VL、MinerU2.5-Pro、Granite-Docling）与语音（Qwen3-ASR）
  • Thinking 模型支持：-Thinking 后缀 handler 自动开启思维模式；strip_thinking 开关剥离 <think> 推理块
  • 无专用 handler 的 GGUF 模型用 llm Model Loader 以通用文本模式运行
  • 无状态推理：每次运行都是独立的一次性请求，不保留跨运行的对话历史

生成控制与资源管理

  • 12 项采样参数：max_tokens、top_k、top_p、min_p、typical_p、temperature、repeat / frequency / present penalty、mirostat 三参
  • n_ctx 上下文长度、vram_limit 显存预算（GB 粒度自动折算 GPU 层数）、image_min/max_tokens 视觉 token 预算、32 位 seed
  • GPU 设备下拉选择：Auto（独显优先、多卡按层切分）或指定单卡整模型加载
  • force_offload 推理后自动卸载 / Unload Model 节点手动卸载 / 前端 Free 按钮与 OOM 处理自动联动卸载
  • 生成中途可取消：点击 ComfyUI Cancel 后推理立即中止（200ms 轮询）
  • 双进度条：ComfyUI 前端进度条 + 终端 tqdm 同步显示（image 逐张模式）；queue_handler 万能输入控制多个推理节点的执行顺序

下游工具链

  • 目标检测流水线：BBox JSON → 像素坐标 → 画框预览图 / SEGS（Impact Pack 兼容）/ MASK（重绘遮罩），支持 dilation 扩张与高斯羽化
  • Parse JSON：剥离代码围栏后解析，点分路径下钻取值，一次输出 any/string/int/float/boolean 五种类型
  • Unpack Code Block：去除输出首尾的 ``` 代码围栏
  • 12 种中文 Prompt Enhancer 预设：Qwen-Image 全系（含 Edit 三代、2512）、Z-Image Turbo、Flux.2 T2I/I2I（输出保持英文）、Wan T2V/I2V/FLF2V
  • 模型目录 models/llm 与 models/LLM 均可识别，支持 extra_model_paths.yaml 挂载外部目录
  • 随 requirements.txt 安装预编译 Vulkan wheel（Windows / Linux ABI3，覆盖 CPython 3.10+）

02用户不可见的功能与机制

不体现在节点 UI 上，但决定插件稳定性与显存行为的内部设计

代码组织（v2.0）

节点与逻辑分层 — app/nodes/ 只放节点声明（文件名一律 node_ 前缀）；跨领域核心逻辑在 app/core/（storage、instruct 基类、devices、handlers 等），纯工具在 app/common/；与单个节点强相关的工具放在该节点文件旁边（bbox_utils.py、prompt_enhancer_presets.py、media 共用的 encoding.py）。

两级 Instruct 基类 — llama_cpp_instruct_base 提供通用骨架（组消息、复制参数、中断监视、收尾清理）与 INPUT_TYPES 字段组装块；llama_cpp_media_instruct_base 叠加 LLAMACPPVLM 端口类型与 mmproj 兜底校验。四个节点子类只声明字段和模态专属的 runner 闭包，各自 40~110 行。

模型生命周期与显存管理

懒加载 — 两个 Loader 节点只做快速失败校验（路径存在、handler 与 mmproj 配对）并输出配置 dict；实际加载推迟到推理节点按需触发。多组 loader + instruct 交错时避免全局单例被反复挤占。

全局单例 LLAMA_CPP_STORAGE — 同一时刻只保留一个模型实例；推理节点每次执行比对 current_config，不一致才重载。配置在加载成功后才记录，失败不会残留导致误判"无需重载"。

先校验后卸载 — load_model 先跑 resolve_config 再清理旧模型，无效配置不会破坏当前已加载的模型。

vram_limit 折算 n_gpu_layers — 手写 GGUF 解析器只读元数据里的 block_count（命中即返回，跳过几十万条 tokenizer 元数据），按文件体积 × 1.55 经验系数均摊出每层显存，mmproj 体积先从预算中扣除。

unload_all_models 猴子补丁 — 启动时替换 comfy.model_management.unload_all_models：前端 Free 按钮或 OOM 处理会顺带卸载 llama 模型与 chat handler。

GPU 设备管理（app/core/devices.py）

ggml C API 枚举设备 — 通过 ctypes 直调 libggml 的 backend 设备接口，区分独显（GPU）与核显（IGPU），完全独立于 PyTorch/CUDA 的设备体系。

只展示可达设备 — 复刻 llama.cpp 收集 model->devices 的规则：只把独显按枚举顺序收入，无独显时才收第一个核显；其余设备 main_gpu 无法选中，因此不出现在下拉框。

选择语义翻译 — Auto = LLAMA_SPLIT_MODE_LAYER（独显优先、多卡按层切分）；显式选卡 = LLAMA_SPLIT_MODE_NONE + 可选列表索引（LAYER 模式下 main_gpu 会被忽略，这是显式选择必须切 NONE 的原因）。

import 时初始化 — 设备枚举在插件加载时同步执行（约几百 ms），属有意设计：UI 下拉框需要在启动期确定设备列表。

多模态输入处理

PNG 无损编码 — 图片统一编码为 PNG base64 而非 JPEG：避免压缩伪影影响 OCR 小字识别，且支持 RGBA。

分辨率策略 — 单帧保持原分辨率；image batch 模式与 video 多帧时 LANCZOS 缩放到 max_size；video 用 linspace 均匀采样并 clamp 到实际帧数，避免重复采样浪费上下文。

音频编码 — ComfyUI AUDIO 均值混为单声道 16-bit WAV（重采样交给 llama.cpp mtmd 端）；音频是否被 mmproj 支持由 llama-cpp-python 侧校验。

prompt 模板替换顺序 — 预设模板先替换 @（按节点 MEDIA_WORD）再注入 #（custom_prompt），避免用户文本中的 @ 被误替换；带 * 的预设强制要求 custom_prompt 填充。

video 系统提示注入 — video Instruct 自动在 system prompt 前追加指令，要求模型把图片序列当作连续视频而非独立静帧。

推理输出与中断

InterruptWatcher 守护线程 — 推理期间每 200ms 轮询 ComfyUI 中断标志，命中调用 Llama.abort()；因 llama-cpp-python 每次请求会 clear abort 事件，命中后持续重复 set 抗竞态。

截断结果丢弃 — abort 会让生成提前返回截断文本，节点检测到中断标志后丢弃结果并抛 InterruptProcessingException 走标准中断流程。

thinking 剥离兼容 — 正则移除 <think>...</think> 块；兼容 generation prompt 已注入开头 <think> 导致输出只含闭合标签的情况，未闭合（生成截断）时保持原样。

hybrid 架构 KV 重置 — 每次请求后用 is_hybrid() / is_recurrent() C API 判断架构，hybrid/recurrent（Qwen3.5、LFM2 系）线性注意力状态无法跨请求前缀复用，命中则整体清空 KV cache 与 hybrid cache；纯 SWA 模型（Gemma3）不受影响。标准 Transformer 交给 llama-cpp-python 的前缀复用机制（批量打标时公共前缀只 prefill 一次）。

handler 注册表 — 对照 llama_multimodal 模块解析 30 个 handler 类，缺失的类打 warning 并从下拉框剔除（防御未来 wheel 升级的类变动）。项目只对接 requirements.txt 固定的 JamePeng Vulkan wheel，不做历史版本兼容，mmproj 统一用 mmproj_path 键传入。

其他基础设施

模型目录注册 — import 时向 folder_paths 注册 llm 与 LLM 两个目录键并绑定 .gguf 扩展名，列表去重合并；模型下拉框按文件名是否含 mmproj 拆分为主模型与 mmproj 两列。

AnyType 万能类型 — AnyType(str) 重载 __ne__ 恒等 False，令 ComfyUI 类型校验对 * 端口放行，实现 Unload Model / queue_handler 的任意类型透传。

cqdm 双进度条 — 一个迭代器同时驱动 ComfyUI 前端 ProgressBar 和终端 tqdm，异常/中断时 finally 收尾避免终端残留。

BBox 绘制与羽化性能 — 画框颜色由 label 内容 MD5 哈希得出（同 label 每次运行同色，80-180 亮度区间保证白字可读）；高斯羽化在 bbox 局部窗口（4 sigma 外扩）上计算，代价不随全图尺寸增长。

CI wheel 构建 — GitHub Actions 手动触发，双平台并行构建 Vulkan ABI3 wheel（一个 wheel 覆盖 CPython 3.10+），开启 GGML_BACKEND_DL + CPU_ALL_VARIANTS 按指令集运行时分发，自动发布 Release。

诊断脚本 — scripts/check_devices.py 可独立运行，列出 GGML 后端检测到的全部设备（CPU/GPU/IGPU/ACCEL），用于排查 Vulkan 环境问题。

已知约束

  • 单模型实例：LLAMA_CPP_STORAGE 为全局单例，不支持同时加载多个模型（多组 loader 交错执行时会互相触发重载）
  • 有独显时核显不可选：llama.cpp 设备收集规则所致；强制核显只能在进程启动前设置 GGML_VK_VISIBLE_DEVICES 环境变量
  • import 时初始化 Vulkan：插件加载耗时约几百毫秒，属有意设计
  • v1.x 旧工作流不兼容：原 Model Loader / Instruct 节点 ID 已被拆分节点取代，加载旧工作流需重建这些节点
